# Documentation Index

This project maintains two documentation surfaces:

- **`website/`** — the VitePress site (guides, API reference, advanced topics, examples). This is the canonical reference for users.
- **`docs/`** — contributor-oriented material (development workflow, tooling, coding standards).

## Contributor Docs

| File | Description |
| --- | --- |
| `DEVELOPMENT.md` | Development workflow, tooling, testing standards, and contributing guidelines. |

## Website

| Section | Path |
| --- | --- |
| Guides | `website/guide/` |
| API Reference | `website/api/` |
| Advanced Topics | `website/advanced/` |
| Examples | `website/examples/` |

## Contributing to Docs

1. For contributor-facing content, edit files in `docs/`.
2. For user-facing content, edit pages in `website/`.
3. Run `npm run docs:dev` from the repo root to preview VitePress changes.
